Frontend folder structure for the Delivery Logs System\n\n- `src/components/`: Reusable UI components (buttons, form fields, modals).\n- `src/pages/`: Route/page components (Dashboard, Deliveries, Login, Admin).\n- `src/services/`: API wrappers and network/service calls (Excel integration, auth).\n- `src/api/`: Low-level request clients (axios instances, interceptors).\n- `src/hooks/`: Reusable React hooks (useAuth, useFetchDeliveries).\n- `src/store/`: Global state (Redux/Context/Pinia stores).\n- `src/utils/`: Utility helpers and formatters.\n- `src/styles/`: Global styles, theme variables.\n- `src/types/`: TypeScript types and interfaces.\n- `src/assets/`: Static assets (images, icons).\n- `src/layouts/`: Layout components (MainLayout, AuthLayout).\n- `src/i18n/`: Localization resources.\n- `src/tests/`: Frontend tests (unit, integration).\n\nNotes:\n- Keep API contracts in `src/services/` and low-level clients in `src/api/`.\n- Put page-level data fetching in `src/pages/` or page-specific hooks.\n
