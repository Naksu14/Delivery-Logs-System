Backend folder structure for the Delivery Logs System (NestJS)\n\n- `src/modules/`: Feature modules (e.g., `deliveries/`, `auth/`, `admin/`). Each module groups controllers, services, dtos.\n- `src/controllers/`: HTTP controllers (routing endpoints).\n- `src/services/`: Business logic services used by controllers and jobs.\n- `src/dtos/`: Data transfer objects and validation classes.\n- `src/entities/`: ORM entities or models (Delivery, Recipient, Package).\n- `src/repositories/`: DB access layer and repository wrappers.\n- `src/config/`: Configuration and environment loaders.\n- `src/migrations/`: DB migration scripts.\n- `src/scripts/`: Utility scripts (excel import/export, seeders).\n- `src/utils/`: Shared helpers (excel-parsing, date helpers).\n- `src/jobs/`: Background jobs and schedulers (notifications, cleanup).\n- `src/database/`: DB connection and adapters (Excel integration helpers).\n- `src/tests/`: Backend tests (unit, integration, e2e helpers).\n\nRecommended module layout (example for `deliveries`):\n- `src/modules/deliveries/deliveries.module.ts`\n- `src/modules/deliveries/deliveries.controller.ts`\n- `src/modules/deliveries/deliveries.service.ts`\n- `src/modules/deliveries/dtos/create-delivery.dto.ts`\n- `src/modules/deliveries/entities/delivery.entity.ts`\n\nNotes:\n- Keep Excel integration encapsulated in `src/scripts/` or `src/services/excel.service.ts`.\n- Store environment-sensitive config in `src/config/` and access via a config service.\n
